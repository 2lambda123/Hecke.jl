    Creation and basics

    Creation of number fields

    Creation of simple number fields

    Number fields are mostly created using the function NumberField, of which number_field is an alias. To create a simple number field, the following functions can be used.

    NumberField(f::Poly{NumFieldElem}, s::String;
            cached::Bool = false, check::Bool = false) -> NumField, NumFieldElem

    Given an irreducible polynomial over some number field , this function creates the simple number field and returns , where is the class of in . The string s is used only for printing the primitive element .

    Testing that is irreducible can be disabled by setting the keyword argument check to false.

    cyclotomic_field(n::Int) -> AnticNumberField, nf_elem

    The -th cyclotomic field defined by the -the cyclotomic polynomial.

    wildanger_field(n::Int, B::fmpz) -> AnticNumberField, nf_elem

    Returns the field with defining polynomial . These fields tend to have non-trivial class groups.

    pure_extension(n::Int, a::NumFieldElem; s = "_$",
               check = true, cached = true) -> NumField, NumFieldElem

    Given an element of a number field and an integer , create the simple extension of with the defining polynomial .

    Many of the constructors have arguments of type Symbol or AbstractString, if used, they define the appearance in printing, and printing only. The named parameter check can be true or false, the default beingtrue`. This parameter controlls is the polynomial defining the number field is tested for irreducibility or not. Given that this can be potentially very time consuming if the degree if large, one can disable this test. Note however, that the behaviour of Hecke is undefined if a reducible polynomial is used to define a field.

    The named boolean parameter cached is inherited from the underlying Nemo system. Two number fields defined using the same polynomial from the identical polynomial ring and the same (identical) symbol/ string will be identical if cached == true and different if cached == false.

    Example

    julia> Qx, x = PolynomialRing(FlintQQ, "x");

julia> K, a = NumberField(x^2 - 10, "a")
(Number field over Rational Field with defining polynomial x^2-10, a)

    Creation of non-simple number fields

    number_field(f::Array{Generic.Poly{T}, 1}, s::String="_\$") where T -> NfRel_ns

    Given polynomials over some number field , construct The ideal in the quotient must be maximal - although this is not tested.

    NumberField(CF::ClassField) -> Hecke.NfRel_ns{Nemo.nf_elem}

    Given a (formal) abelian extension, compute the class field by finding defining polynomials for all prime power cyclic subfields. Note, by type this is always a non-simple extension.

    Example

    julia> Qx, x = PolynomialRing(FlintQQ, "x");

julia> K, g = number_field([x^2-2, x^2-3, x^2-5])
(Non-simple number field with defining polynomials fmpq_mpoly[_$1^2 - 2, _$2^2 - 3, _$3^2 - 5], NfAbsNSElem[_$1, _$2, _$3])

julia> g[1]^2
2

julia> minpoly(g[1] + g[2])
x^4-10*x^2+1
